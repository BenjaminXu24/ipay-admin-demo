# IPAYS 后台 PRD

## 1. 文档目标

本 PRD 用于说明 IPAYS 管理后台中“商户充值/提现申请与审核”需求的页面结构、字段定义、交互规则、状态流转、异常场景与验收标准，供产品、前端、后端、测试统一理解和落地。

## 2. 需求背景

当前商户充值、提现在发起后直接变更商户账户余额，缺少审核链路与过程留痕，存在以下问题：

- 发起后无法区分“已申请”与“已生效”
- 财务与老板缺少明确分工的审核入口
- 发起人无法查看当前申请状态
- 被拒绝、打回、生效失败时缺少原因反馈
- 存在重复操作和口头确认带来的风险

本次需求将商户充值/提现改为“申请制 + 初审 + 复审”模式，并补齐申请记录查看能力，使整条链路可见、可追踪、可留痕。

## 3. 需求目标与用户价值

### 3.1 需求目标

在商户账户充值/提现操作中增加审核机制，并补齐发起人查看申请记录与审核状态的入口，形成完整闭环。

### 3.2 用户价值

对发起人：

- 知道申请是否提交成功
- 知道当前停留在初审还是复审
- 知道是否已生效
- 知道被拒绝、打回或生效失败的原因
- 减少重复提交与反复追问

对平台内部：

- 降低沟通成本
- 降低误操作风险
- 提高审核流转透明度
- 增强财务操作可追踪性

## 4. 模块范围

### 4.1 涉及页面

- 商户详情页 - 支付账户管理
- 商户申请记录页
- 审核管理 - 初审列表
- 审核管理 - 复审列表

### 4.2 不变范围

本次不改动以下内容：

- 商户详情页中充值/提现入口位置
- 发起充值/提现的两步操作形式
  - 第一步：填写申请资料
  - 第二步：短信验证码验证

改动点仅为：通过验证码后不再直接变更金额，而是生成申请单进入审核池。

## 5. 页面结构

### 5.1 页面 A：商户详情页 - 支付账户管理

#### 页面目标

1. 展示商户当前账户余额与流水
2. 提供充值/提现发起入口
3. 提供申请记录查询入口

#### 页面模块

1. 余额展示区
2. 操作按钮区
   - 充值
   - 提现
   - 申请记录
3. 账户流水查询区
4. 账户流水列表区

#### 关键规则

- `申请记录` 按钮与 `充值`、`提现` 同级展示
- 点击充值/提现后，仍保持原有两步弹框流程
- 短信验证通过后，生成申请单
- 商户账户余额在申请阶段不变

### 5.2 页面 B：申请记录页

#### 页面目标

用于查看当前商户下所有充值/提现申请记录及其状态，仅承担查询与查看职责，不承担审核动作。

#### 页面模块

1. 页面标题区
2. 当前商户信息区
3. 查询筛选区
4. 申请列表区
5. 分页区

#### 详情查看模块

1. 基本信息
2. 发起信息
3. 发起凭证
4. 初审结果
5. 收款/付款凭证
6. 复审结果
7. 最终结果
8. 流转记录

#### 关键规则

- 从商户详情页进入时，默认固定为当前商户维度
- 只读查看，不允许审核、编辑、删除
- 已终止、已生效、生效失败的申请也仅允许查看

### 5.3 页面 C：审核管理 - 初审列表

#### 页面目标

供财务查看待初审申请并执行初审。

#### 页面模块

1. 查询筛选区
2. 待处理列表区
3. 初审详情/弹框

#### 角色说明

- 财务角色使用
- 处理初审通过或初审拒绝

### 5.4 页面 D：审核管理 - 复审列表

#### 页面目标

供老板查看待复审申请并执行复审。

#### 页面模块

1. 查询筛选区
2. 待处理列表区
3. 复审详情/弹框

#### 角色说明

- 老板角色使用
- 处理复审通过或复审打回

## 6. 主流程

### 6.1 主流程一：发起申请

1. 在商户详情页进入支付账户管理
2. 点击 `充值` 或 `提现`
3. 填写申请资料：
   - 交易金额
   - 交易日期
   - 交易证明文件
   - 收款渠道
   - 收款账号
   - 备注
4. 点击下一步，进入短信验证码验证
5. 验证通过后生成申请单
6. 当前状态为 `待初审`
7. 商户账户余额保持不变

### 6.2 主流程二：发起人查看申请记录

1. 在支付账户管理页点击 `申请记录`
2. 进入申请记录页
3. 查看当前商户下所有充值/提现申请记录
4. 点击 `查看`
5. 查看当前状态、发起资料、初审结果、复审结果、最终结果与流转记录

### 6.3 主流程三：财务初审

1. 财务进入 `初审列表`
2. 查看待处理申请
3. 点击 `审核`
4. 查看发起资料与发起凭证
5. 执行初审：
   - 初审通过：
     - 上传收款/付款凭证
     - 可填写处理备注
     - 提交后状态变为 `初审通过，待复审`
   - 初审拒绝：
     - 填写拒绝原因
     - 提交后状态变为 `初审拒绝，已终止`

### 6.4 主流程四：老板复审

1. 老板进入 `复审列表`
2. 查看待复审申请
3. 点击 `审核`
4. 查看发起资料、发起凭证、初审结果、收款/付款凭证
5. 执行复审：
   - 复审通过：
     - 状态先进入 `复审通过，生效中`
     - 系统执行实际充值/提现
     - 成功后变为 `复审通过，已生效`
   - 复审打回：
     - 填写打回原因
     - 状态变为 `复审打回，待初审`
     - 返回初审列表再次处理

## 8. 状态清单与流转

### 8.1 业务状态

统一状态如下：

- 待初审
- 初审通过，待复审
- 初审拒绝，已终止
- 复审打回，待初审
- 复审通过，生效中
- 复审通过，已生效
- 复审通过，生效失败

### 8.2 状态流转表

| 当前状态 | 操作 | 结果状态 |
| --- | --- | --- |
| 待初审 | 初审通过 | 初审通过，待复审 |
| 待初审 | 初审拒绝 | 初审拒绝，已终止 |
| 初审通过，待复审 | 复审通过 | 复审通过，生效中 |
| 初审通过，待复审 | 复审打回 | 复审打回，待初审 |
| 复审打回，待初审 | 再次初审通过 | 初审通过，待复审 |
| 复审通过，生效中 | 系统执行成功 | 复审通过，已生效 |
| 复审通过，生效中 | 系统执行失败 | 复审通过，生效失败 |

## 10. 关键规则与校验

### 10.1 发起申请规则

- 验证码验证通过前，不生成申请单
- 申请提交后，不直接变更商户余额
- 申请提交成功后，状态必须为 `待初审`

### 10.2 申请记录权限

建议口径：

- 有充值/提现发起权限的账号，必须同时拥有申请记录查看权限
- 从商户详情进入申请记录时，默认查看当前商户下全部申请记录
- 该页面仅用于查看，不具备审核权限

### 10.3 初审规则

- 初审通过时，必须上传平台凭证图片
  - 充值申请：平台收款凭证
  - 提现申请：平台付款凭证
- 初审拒绝时，拒绝原因必填
- 初审提交时需校验当前申请状态，若状态已变化则提示：
  - `当前申请状态已更新，请刷新后重试`

### 10.4 复审规则

- 复审通过后，系统才执行实际充值/提现
- 复审打回时，打回原因必填
- 复审提交时需校验当前申请状态，若状态已变化则提示：
  - `当前申请状态已更新，请刷新后重试`

### 10.5 凭证展示规则

- 发起凭证、平台凭证均以图片形式展示
- 初审上传平台凭证时，使用图片上传位，不使用普通文本输入框
- 凭证缺失时需展示空态，不可直接显示为空字符串

### 10.6 多轮打回规则

- 同一申请允许出现“复审打回 -> 再次初审 -> 再次复审”的循环
- 详情页必须保留完整流转记录
- 列表页仅展示当前最新状态

## 11. 异常场景

### 11.1 发起提交失败

场景：

- 必填项未填
- 短信验证码失败
- 凭证上传失败
- 接口异常

处理：

- 不生成申请单
- 保持在当前弹框
- 明确提示失败原因

### 11.2 初审通过但未上传平台凭证

处理：

- 不允许提交
- 提示必须补齐凭证

### 11.3 初审拒绝或复审打回未填写原因

处理：

- 不允许提交
- 明确提示原因必填

### 11.4 复审通过后执行失败

场景：

- 商户余额不足
- 渠道异常
- 系统异常

处理：

- 状态更新为 `复审通过，生效失败`
- 申请记录页与详情页同时展示失败原因
- 不允许错误显示为“已生效”

### 11.5 同一申请被重复审核

场景：

- 两个审核人同时打开同一申请
- 一人先提交，另一人后提交

处理：

- 提交前校验当前状态
- 若状态已变化，提示：
  - `当前申请状态已更新，请刷新后重试`

### 11.6 商户范围错误

处理：

- 从商户详情进入申请记录页时，必须自动带入当前商户
- 禁止在 A 商户页面看到 B 商户申请记录

## 13. 验收标准

1. 商户详情页的支付账户管理中，`充值`、`提现` 按钮后新增 `申请记录` 按钮。
2. 发起充值或提现申请后，商户余额不立即变化。
3. 验证码通过后生成申请单，状态为 `待初审`。
4. 从支付账户管理点击 `申请记录` 后，默认只展示当前商户的申请记录。
5. 申请记录页支持查看申请详情，且详情默认只读。
6. 初审列表支持按 `当前状态` 筛选。
7. 复审列表支持按 `当前状态` 筛选。
8. 初审通过时必须上传平台凭证图片，否则不可提交。
9. 初审拒绝时必须填写拒绝原因，否则不可提交。
10. 复审打回时必须填写打回原因，否则不可提交。
11. 复审通过后，系统才执行实际充值/提现。
12. 执行成功时，申请状态更新为 `复审通过，已生效`，商户余额同步变化。
13. 执行失败时，申请状态更新为 `复审通过，生效失败`，并展示失败原因。
14. 多次打回、再次初审的申请，在详情页流转记录中可完整查看历史处理过程。
15. 已终止、已生效、生效失败的申请均不可再次审核，只可查看。
