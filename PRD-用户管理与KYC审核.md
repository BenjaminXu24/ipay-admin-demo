# IPAYS 后台 PRD

## 1. 文档目标

本 PRD 用于说明 IPAYS 管理后台中“用户管理”与“用户认证审核管理”两个模块的页面结构、字段定义、交互规则、状态流转与验收标准，供前端、后端、测试共同理解和落地。

## 2. 模块范围

- 一级菜单：用户管理
- 二级菜单：
  - 用户管理
  - 用户认证审核管理

## 3. 页面一：用户管理

### 3.1 页面目标

用于查看和管理即将开通或已开通 IPAY 钱包的用户信息，包括基础资料、当前钱包等级、钱包余额、用户状态与 KYC 生效资料。

### 3.2 列表字段

| 字段 | 说明 | 是否可编辑 | 规则 |
| --- | --- | --- | --- |
| 用户ID | 系统唯一用户标识 | 否 | 唯一 |
| 手机号 | 菲律宾手机号 | 否 | 列表展示，详情可编辑；仅支持 `9xxxxxxxxx` 或 `09xxxxxxxxx` |
| 用户姓名 | 用户实名 | 否 | 列表展示，详情可编辑 |
| 钱包等级 | 当前钱包等级 | 否 | 由 KYC 审核通过或系统规则驱动，不在资料页直接编辑 |
| KYC状态 | 最近一次 KYC 申请审核状态 | 否 | 待审核 Pending Review / 审核通过 Approved / 审核拒绝 Rejected |
| 钱包余额 | 当前钱包余额 | 否 | 金额展示，单位 PHP |
| 用户状态 | 用户状态 | 否 | 列表展示，详情页可做少量受控调整 |
| 最近更新时间 | 最近一次资料或审核更新时间 | 否 | 日期时间 |
| 操作 | 查看 | 否 | 进入详情页 |

### 3.3 筛选条件

- 用户ID
- 手机号
- 钱包等级
- 状态
- 用户姓名
- 证件类型

### 3.4 详情页结构

详情页包含两个 Tab：

1. 资料
2. 钱包

### 3.5 详情页 - 资料 Tab

#### 字段范围

- 用户ID
- 手机号
- 状态
- 用户姓名
- 出生日期
- 国籍
- 电子邮件
- 家庭地址
- 证件类型
- 证件号码
- 证件照片
- 人脸照片

#### 规则说明

- 用户ID只读
- 钱包等级只读，不在该 Tab 直接编辑
- 手机号需校验菲律宾手机号格式
- 邮箱需校验邮箱格式
- 点击“保存”后提示保存成功
- 保存成功后当前页信息刷新
- 页面需展示“当前生效信息”和“KYC申请记录”两个区块
- 页面需展示用户操作留痕

### 3.6 详情页 - 钱包 Tab

#### 展示内容

- 钱包余额
- 钱包明细列表

#### 钱包明细字段

- 类型
- 渠道
- 交易金额
- 创建时间
- 备注
- 余额

#### 交互规则

- 当前阶段钱包页仅展示余额与流水
- 不提供充值、提现入口
- 充值/提现属于后续单独需求，不作为本期用户详情页能力

## 4. 页面二：用户认证审核管理

### 4.1 页面目标

用于审核用户从 App 端提交的钱包 KYC 认证资料。

### 4.2 审核范围

- Basic 不需要审核
- 仅审核：
  - Verified
  - Fully Verified

### 4.3 列表字段

| 字段 | 说明 |
| --- | --- |
| 申请单号 | 本次 KYC 申请唯一编号 |
| 用户ID | 用户唯一标识 |
| 用户姓名 | 用户实名 |
| 手机号 | 用户手机号 |
| 当前等级 | 申请前钱包等级 |
| 申请等级 | 本次申请目标等级 |
| 证件类型 | 用户提交的证件类型 |
| 证件号码 | 用户提交的证件号码 |
| 人脸照片 | 是否上传人脸照片 |
| 提交时间 | 提交申请时间 |
| 审核状态 | 待审核 Pending Review / 审核通过 Approved / 审核拒绝 Rejected |
| 审核备注 | 审核说明 |
| 操作 | 审核 / 查看 |

### 4.4 审核详情/弹窗内容

#### 基础信息

- 申请单号
- 用户
- 当前等级
- 申请等级
- 证件类型
- 证件号码
- 手机号
- 提交时间

#### 照片区域

- 证件照片展示区
- 人脸照片展示区

#### 照片规则

- 证件照片为必填，必须预留展示区
- 人脸照片为选填，即使用户未上传，也必须保留展示区并展示“未上传”空态
- 审核人员可通过证件照片和人脸照片进行人工比对

#### 审核操作

- 审核结果：
  - 通过
  - 拒绝
- 审核通过后等级（只读展示，由申请等级决定）：
  - Verified 申请 -> Verified
  - Fully Verified 申请 -> Fully Verified
- 审核备注
- 审核人
- 审核时间
- 审核留痕记录

### 4.5 审核规则

- 点击“确认提交”后保存审核结果
- 若审核结果为“通过”，则同步更新该用户的钱包等级
- 若审核结果为“拒绝”，必须填写拒绝原因
- 审核通过/拒绝后需记录审核人、审核时间、审核备注
- 已审核记录再次打开时仅可查看，不可重复提交
- Enhanced 不在普通 KYC 审核流程中处理

## 5. 钱包等级说明

| 等级 | 说明 |
| --- | --- |
| Basic | 基础钱包，无需审核 |
| Verified | 用户提交基础 KYC 资料后审核通过 |
| Fully Verified | 用户提交更完整的 KYC 资料后审核通过 |
| Enhanced | 增强账户，由系统规则或后台受控操作单独处理，不走普通 KYC 审核 |

## 6. 状态流转

### 6.1 用户状态

| 当前状态 | 可操作 | 结果状态 |
| --- | --- | --- |
| 启用 | 禁用 / 注销 | 禁用 / 注销 |
| 禁用 | 启用 / 注销 | 启用 / 注销 |
| 注销 | 无 | 注销 |

### 6.2 KYC 审核状态

| 当前状态 | 操作 | 结果 |
| --- | --- | --- |
| 待审核 | 通过 | 更新钱包等级并记录备注 |
| 待审核 | 拒绝 | 保持原等级并记录拒绝原因 |
| 通过 | 查看 | 只读查看 |
| 拒绝 | 查看 | 只读查看 |

## 7. 异常场景

- 用户未上传人脸照片时，页面仍需展示预留区域，文案提示“未上传”
- 图片加载失败时，前端需展示占位图或失败提示
- 注销状态用户不可继续进行钱包等级提升
- 审核通过后如同步用户等级失败，应提示“审核结果保存失败”
- 手机号或邮箱格式错误时不可保存用户资料
- 审核拒绝但未填写原因时不可提交

## 9. 验收标准

1. 左侧菜单中可看到“用户管理”一级菜单及两个二级菜单。
2. 用户管理列表中展示轻列表字段：用户ID、手机号、用户姓名、钱包等级、KYC状态、钱包余额、用户状态、最近更新时间。
3. 点击“查看”可进入用户详情页。
4. 用户详情页包含“资料”和“钱包”两个 Tab。
5. 资料 Tab 中修改手机号或邮箱格式错误时不可保存。
6. 钱包 Tab 不展示充值、提现按钮。
7. 用户认证审核列表中可查看所有 Verified 与 Fully Verified 申请记录。
8. 审核详情中必须展示证件照片区域。
9. 审核详情中必须预留人脸照片区域，未上传时展示空态。
10. 审核通过后，用户钱包等级按申请等级同步更新。
11. 审核拒绝时，未填写拒绝原因不可提交。
12. KYC 详情中展示审核人、审核时间和审核留痕。
