# IPAYS 后台 PRD

## 1. 文档目标

本 PRD 用于说明 iPay 后台“资金总览”页面的页面结构、展示口径、交互规则、异常场景与验收标准，供产品、前端、后端、测试共同理解和落地。

## 2. 需求背景

当前 iPay 对接 Netbank、StarPay、Coins 多个平台。财务若要查看各平台余额，需要分别登录不同平台进行查询，操作繁琐，不利于日常资金查看与对账准备。

同时，老板希望在 iPay 后台中统一看到：

- iPay 总余额
- 每个支付平台余额
- 每日资金流入与流出汇总

因此，需要在后台新增 `资金总览` 页面，统一查看多平台余额与每日资金汇总。

## 3. 需求目标

### 3.1 业务目标

提供统一入口，减少多平台切换，提高财务日常查看效率与对账准备效率。

### 3.2 页面目标

页面需完成两项核心任务：

1. 展示 Netbank、StarPay、Coins 当前实际余额及 iPay 总余额
2. 展示按天汇总的资金流入 / 流出 / 净额、分类统计及手续费影响

## 4. 页面结构

页面从上到下分为四个区域：

1. 页面标题与说明区
2. 余额总览区
3. 查询筛选区
4. 每日资金汇总区

## 6. 关键业务口径

### 6.1 平台余额口径

平台余额展示的是接口返回的当前实际余额：

- Netbank 实际余额
- StarPay 实际余额
- Coins 实际余额

### 6.2 iPay 总余额口径

iPay 总余额 = Netbank余额 + StarPay余额 + Coins余额

#### 特殊规则

若任意一个平台余额查询失败：

- iPay 总余额显示 `--`
- 失败平台不按 0 参与合计
- 提示文案统一为：`查询失败，请重新查询`

### 6.3 每日流入口径

每日总流入 = 支付业务流入 + 商户手续费收入 + 人工充值

其中：

- 支付业务流入 = Collection 成功订单金额汇总
- 商户手续费收入 = Collection 商户手续费 + Payout 商户手续费
- 人工充值 = 商户人工充值成功入账金额汇总

### 6.4 每日流出口径

每日总流出 = 支付业务流出 + 平台手续费支出 + 人工提现

其中：

- 支付业务流出 = Payout 成功订单金额汇总
- 平台手续费支出 = Collection 平台手续费 + Payout 平台手续费
- 人工提现 = 商户人工提现成功出账金额汇总

### 6.5 净额口径

净额 = 总流入 - 总流出

### 6.6 统计状态口径

每日汇总仅统计成功状态数据，不统计：

- 失败
- 处理中
- 已取消
- 已拒绝

### 6.7 手续费业务说明

手续费分为两类：

#### 商户手续费

- 每笔订单向商户收取的手续费
- Payout 场景示例：
  - 用户放款 100
  - 商户手续费 10
  - 商户账户实际扣减 110
- Collection 场景示例：
  - 用户支付 100
  - 商户手续费 10
  - 商户最终结算到账 90

在资金统计中，商户手续费计入 `商户手续费收入`。

#### 平台手续费

- 每笔交易支付给支付平台的手续费
- 平台手续费对象是渠道方，不是商户
- 在资金统计中，平台手续费计入 `平台手续费支出`

## 7. 主流程

### 7.1 进入页面查看余额

1. 用户进入 `资金总览`
2. 页面自动查询 Netbank、StarPay、Coins 当前余额
3. 成功时展示各平台余额与更新时间
4. 三平台均成功时展示 iPay 总余额

### 7.2 手动刷新余额

1. 用户点击 `刷新余额`
2. 页面重新查询三平台余额
3. 更新各平台余额与 iPay 总余额

### 7.3 查看默认每日汇总

1. 页面进入后默认加载最近 N 天数据
2. 列表展示每日总流入、总流出、净额与分类统计

### 7.4 按日期范围查询

1. 用户选择开始日期与结束日期
2. 点击 `查询`
3. 页面刷新对应日期范围内的每日资金汇总

### 7.5 导出

1. 用户设置日期范围
2. 点击 `导出`
3. 导出当前查询条件下结果

## 8. 异常场景

### 8.1 某个平台余额查询失败

- 失败平台余额显示 `--`
- 对应卡片展示 `查询失败，请重新查询`
- 其他成功平台正常展示
- iPay 总余额显示 `--`

### 8.2 全部平台余额查询失败

- 所有平台余额显示 `--`
- iPay 总余额显示 `--`
- 页面提示 `查询失败，请重新查询`

### 8.3 日期范围非法

包括：

- 开始日期为空
- 结束日期为空
- 开始日期晚于结束日期
- 日期跨度超过最大限制

处理：

- 不允许查询
- 提示用户修正

### 8.4 查询范围内无数据

- 汇总区显示空态
- 文案：`当前时间范围内暂无数据`

### 8.5 汇总查询失败

- 汇总区显示错误态
- 支持重新查询
- 不影响余额卡片展示

### 8.6 无权限访问

- 菜单不展示或不可进入
- 即使进入，页面展示 no permission
- 后端接口需同步拦截

## 11. 关键规则与校验

### 11.1 余额刷新规则

- 页面打开时自动查询一次余额
- 点击 `刷新余额` 时仅刷新余额区

### 11.2 汇总查询规则

- 查询每日汇总时仅刷新汇总区
- 不影响余额区

### 11.3 日期校验规则

- 开始日期必填
- 结束日期必填
- 开始日期不能晚于结束日期
- 建议限制最大跨度 90 天

### 11.4 展示规则

- 平台余额失败时显示 `--`
- 总余额只要有一个平台失败，也显示 `--`
- 失败提示统一使用：`查询失败，请重新查询`

### 11.5 币种规则

- 本次统一为 PHP
- 不处理多币种

### 11.6 权限规则

建议独立配置：

- 资金总览查看权限
- 刷新余额权限
- 导出权限

## 12. 验收标准

1. 后台新增 `资金总览` 页面入口。
2. 页面打开时自动查询 Netbank、StarPay、Coins 当前余额。
3. 页面展示各平台余额与更新时间。
4. 页面展示 iPay 总余额。
5. 任意一个平台查询失败时，对应平台余额显示 `--`。
6. 任意一个平台查询失败时，iPay 总余额显示 `--`。
7. 页面统一展示提示文案：`查询失败，请重新查询`。
8. 点击 `刷新余额` 时仅刷新余额区。
9. 页面默认加载每日资金汇总数据。
10. 支持按开始日期、结束日期查询每日汇总。
11. 每日汇总表至少展示日期、总流入、总流出、净额、支付业务流入、支付业务流出、人工充值、人工提现。
12. 每日汇总需额外展示商户手续费收入、平台手续费支出。
13. 查询无数据时展示空态：`当前时间范围内暂无数据`。
14. 日期非法时不允许查询，并提示用户修正。
15. 总流入 / 总流出 / 净额计算口径正确。
16. 每日汇总仅统计成功状态数据。
